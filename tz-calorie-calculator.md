# ТЗ: Калькулятор калорий для кафе

## Краткое описание

Одностраничное веб-приложение (SPA) — интерактивный калькулятор калорий для кафе в Польше. Гость сканирует QR-код на столе или переходит по ссылке из Instagram, видит меню с категориями, добавляет блюда и получает суммарные БЖУ и калории выбранных позиций.

Референс: https://sagemoscow.ru/nutrition

---

## Целевая аудитория и сценарий использования

- **Кто:** гости кафе (Польша), преимущественно мобильные пользователи
- **Как попадают:** QR-код на столах, ссылка в Instagram bio
- **Что делают:** открывают страницу → выбирают блюда → видят суммарную калорийность и БЖУ
- **Язык интерфейса:** только польский

---

## Функциональные требования

### 1. Отображение меню

- Блюда сгруппированы по категориям (например: Przystawki, Zupy, Dania główne, Desery и т.д.)
- Категории можно переключать табами или аккордеоном (mobile-first)
- Каждое блюдо отображает:
  - Название
  - Белки (B), Жиры (T), Углеводы (W) — на порцию
  - Калорийность (kcal) — на порцию
  - Кнопка «+» для добавления

### 2. Добавление / удаление блюд

- Кнопка «+» добавляет 1 порцию блюда в подсчёт
- При повторном нажатии — увеличивает количество (счётчик: −/число/+)
- Кнопка «−» уменьшает количество; при 0 — блюдо убирается из подсчёта
- Визуальная индикация выбранных блюд (подсветка или бейдж)

### 3. Итоговая панель

- Фиксированная внизу экрана (sticky footer)
- Показывает суммарные значения выбранных блюд:
  - Białka (B): XX g
  - Tłuszcze (T): XX g
  - Węglowodany (W): XX g
  - Kalorie: XXX kcal
- Кнопка «Wyczyść» (сбросить выбор)
- Обновляется в реальном времени при каждом добавлении/удалении

### 4. Дополнительно (nice to have, не MVP)

- Поиск/фильтр по названию блюда
- Отображение аллергенов (если данные есть)
- Возможность поделиться выбором (share-ссылка или скриншот)

## Нефункциональные требования

### Дизайн и UX

- **Mobile-first** — основной трафик с телефонов
- Адаптивная вёрстка: корректно выглядит на экранах 320px–1440px
- Быстрая загрузка: целевой LCP < 2 секунд
- Минималистичный, современный дизайн (стиль: светлая тема, чистая типографика)
- Никаких внешних зависимостей на этапе загрузки (шрифты — системные или загружаемые асинхронно)

### Производительность

- Статический сайт, без серверного рендеринга
- Вся логика на клиенте (JS)
- Данные меню загружаются из JSON-файла, встроенного в бандл или подгружаемого при загрузке

## Техническая архитектура

### Стек

- **Фреймворк:** React (Vite как сборщик)
- **Стилизация:** Tailwind CSS
- **Хостинг:** Cloudflare Pages (бесплатный план)
- **Домен:** .com через Cloudflare Registrar (~$10.46/год) или .pl через внешнего регистратора (nazwa.pl, OVH.pl) с DNS на Cloudflare

### Структура проекта

```
/
├── public/
│   └── favicon.ico
├── src/
│   ├── data/
│   │   └── menu.json          ← данные меню (единственный файл для обновления)
│   ├── components/
│   │   ├── App.jsx
│   │   ├── CategoryTabs.jsx   ← переключатель категорий
│   │   ├── MenuList.jsx       ← список блюд в категории
│   │   ├── MenuItem.jsx       ← карточка блюда с кнопкой +/−
│   │   └── SummaryBar.jsx     ← sticky итоговая панель
│   ├── hooks/
│   │   └── useCart.js          ← логика подсчёта (state + калькуляция)
│   ├── index.css
│   └── main.jsx
├── index.html
├── package.json
├── tailwind.config.js

```

### Формат данных menu.json

```json
{
  "categories": [
    {
      "id": "przystawki",
      "name": "Przystawki",
      "items": [
        {
          "id": "bruschetta",
          "name": "Bruschetta z pomidorami",
          "proteins": 5.2,
          "fats": 8.1,
          "carbs": 22.4,
          "kcal": 185
        }
      ]
    },
    {
      "id": "zupy",
      "name": "Zupy",
      "items": [...]
    }
  ]
}
```

**Поля каждого блюда:**
| Поле | Тип | Описание |
|------|-----|----------|
| id | string | Уникальный идентификатор |
| name | string | Название на польском |
| proteins | number | Белки, грамм на порцию |
| fats | number | Жиры, грамм на порцию |
| carbs | number | Углеводы, грамм на порцию |
| kcal | number | Калорийность, kcal на порцию |

### Обновление меню

Для изменения меню нужно:
1. Отредактировать файл `src/data/menu.json`
2. Закоммитить и запушить в Git-репозиторий
3. Cloudflare Pages автоматически соберёт и задеплоит новую версию

---

## Деплой

### Первичная настройка

1. Создать репозиторий на GitHub
2. Подключить его к Cloudflare Pages (dashboard → Pages → Create a project → Connect to Git)
3. Указать настройки сборки:
   - Build command: `npm run build`
   - Build output directory: `dist`
4. Привязать домен в настройках Cloudflare Pages

### Последующие обновления

- `git push` в main → автоматический деплой за ~30 секунд
